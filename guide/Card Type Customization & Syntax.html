Anki Syntax

Syntax Explanation 
{{Field Name}}
 Gets replaced with whatever is in the field with the specified name. It’s raw, so if a field said “Hello World”, it would literally get replaced with just Hello World in the code. This means you can also use it in JavaScript, although be sure to surround it with quotes first.

Also, any audio files or images placed in fields will be replaced with their HTML counterparts, so you don’t have to worry about formatting those (although you might want to - try it and see).**		 
{{#Field Name}}
     <!-- CODE HERE -->
{{/Field Name}}
 Basically, “if there is anything in this field, include this code”. If it’s empty, the entirety of the block effectively gets erased. If it’s not, the {{}} parts are erased, but the code in between them remains. This can be used anywhere, including in JavaScript. 
{{^Field Name}}
     <!-- CODE HERE -->
{{/Field Name}}
 The counterpart to the previous one. Basically, “if there is nothing in this field, include this code”. If it’s not empty, the entirety of the block effectively gets erased. If it is, the {{}} parts are erased, but the code in between them remains. This can be used anywhere, including in JavaScript.

The previous two can be “combined”, one after another, to effectively create an “if not empty, else” statement.		 
{{FrontSide}}
 99% of the time, this should be the first line on the back sides of all of your cards. It copies everything from the front, EXCEPT the next thing: 
{{type:Field Name}}
 Creates a text field for the user to type into when studying. Whatever they type will be checked against whatever is in the specified field. The results of this check will be displayed where the text field is, or can be customized with the next thing: 
<hr id=answer>
 Just a horizontal rule with the id “answer”. For cards with the previous thing (type fields) on the front, Anki looks for these on the back to display the comparison between the user’s response and what was in the field.

Keep in mind these do not automatically “score” the card for you. You still have to click one of the buttons to tell Anki how you did.		 
<script>document.documentElement.setAttribute("lang", "ja");</script>
 Not Anki-specific, but just a one-liner that is a must-have for Japanese studying. Put it at the very top of the front sides of all of your cards.

Explanation: If you’ve ever seen weird looking kanji when reading Japanese text on the computer, that’s because most East-Asian Hanzi (called kanji in Japanese), despite having slight variations, all have the same unicode values. It’s up to the computer to determine which language they’re being presented in, and display the right version. Most things default to Chinese, so if you’re not careful, you’ll end up learning those versions instead of the Japanese versions. This forces the card to display the Japanese versions.

tldr: If you don’t do this, kanji will look wrong.

There are a few more, but I’ve never needed to use them. You can find them on the Anki docs, though.

Tips

Conditional Cards

Anki doesn’t generate cards with empty front sides. This means that, for example, if you wanted to make a card that only showed if the word had kanji, you could surround literally everything on the front with {{#Kanji}} and {{/Kanji}}.

Audio

To use audio, adding the {{Field Name}} tag will put an audio play button in that location. Click here for information about adding audio files to fields.
